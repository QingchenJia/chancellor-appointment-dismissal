# 宋代宰辅任免查询系统设计

## 背景

项目当前只有原始数据文件 `宋代宰辅编年录.xlsx`。该工作簿记录两宋宰相与执政官员的任免、调整、任期状态和部分日期校订说明。

主表结构为宽表：

- 行表示时间，起自建隆元年二月，终于祥兴二年二月。
- A-D 列分别为公元、皇帝、年号、月份。
- E 列以后每列为一位宰执人物。
- 人物列中的非空单元格记录任免事件或任期状态。
- 批注中保存日期校订、史源差异和说明。
- 任职至罢职期间还通过边框样式辅助表达任期区间。

目标是将宽表拆分为符合数据库范式的结构，并实现一个本地轻量 Web 查询系统。用户应能按年月、人名、皇帝、年号、任免类型和原文关键词查询任命、罢免、调整等职务变动，并能追溯到 Excel 原始文本与源单元格。仅表示延续在任、没有职务变动的任期状态记录不进入数据库。

## 交付形态

采用轻量服务版：

- 后端：FastAPI。
- 数据库：SQLite。
- 前端：单页 Web 应用。
- 数据导入：使用 `document` conda 环境中的 Python 和 `openpyxl` 读取 Excel。

该方案兼顾规范化数据设计、查询能力、实现成本和本地运行便利性。当前不引入 PostgreSQL/MySQL，也不采用纯静态 JSON 查询。

## 数据模型

### `persons`

人物表。

字段：

- `id`
- `canonical_name`
- `raw_name`
- `aliases`
- `source_column`

用途：

- 保存标准姓名。
- 保留 Excel 表头原文。
- 对“张旻（张耆）”等带别名表头进行半结构化拆分。

### `time_points`

时间点表，每条记录对应 Excel 主表的一行年月。

字段：

- `id`
- `gregorian_year`
- `month_index`
- `month_label`
- `emperor`
- `era_name`
- `source_row`

规则：

- 以原表年月为主，不强行把农历日换算为公历日期。
- 皇帝、年号、公元年如在 Excel 中以空白延续，应在导入时继承上一条有效值。

### `appointment_events`

任免、调整、死亡等职务变动事件表。

字段：

- `id`
- `person_id`
- `time_point_id`
- `event_type`
- `raw_text`
- `source_cell`
- `is_event_text`
- `is_tenure_state`
- `parse_confidence`

规则：

- Excel 中每个非空人物单元格先进行分类；仅职务变动事件生成记录。
- 包含“本月、除、以、拜、罢、免、落、卒、死难、赴水死”等动作词的记录标记为事件文本。
- 仅包含官职名称、没有明显动作词的任期状态记录在导入阶段跳过。
- 自动判断只作为检索辅助，原文始终保留。

### `annotations`

批注表。

字段：

- `id`
- `event_id`
- `source_cell`
- `comment_text`

用途：

- 保存 Excel 批注。
- 支持详情页展示日期校订与史源说明。

### `import_audit`

导入审计表。

字段：

- `id`
- `source_file`
- `imported_at`
- `row_count`
- `column_count`
- `person_count`
- `record_count`
- `comment_count`
- `warning_count`

用途：

- 记录导入统计。
- 便于核对原始 Excel 与数据库记录数量。

## 导入流程

导入命令：

```bash
conda run -n document python scripts/import_excel.py 宋代宰辅编年录.xlsx --db data/song_chancellors.db
```

重建数据库：

```bash
conda run -n document python scripts/import_excel.py 宋代宰辅编年录.xlsx --db data/song_chancellors.db --rebuild
```

导入步骤：

1. 使用 `openpyxl` 打开工作簿。
2. 读取主表第 2 行，建立人物列到 `persons` 的映射。
3. 从第 3 行开始读取时间行，继承空白的公元、皇帝、年号字段。
4. 对每个非空人物单元格分类，跳过任期状态文本，对职务变动文本生成 `appointment_events` 记录。
5. 读取批注并写入 `annotations`。
6. 记录导入统计和解析 warning。

导入不因单个单元格解析失败而中断。失败或低置信度解析应记录 warning，并保留原始文本。

## 后端 API

### `GET /api/search/events`

综合检索职务变动事件。

参数：

- `year_from`
- `year_to`
- `month`
- `person`
- `event_type`
- `emperor`
- `era`
- `keyword`
- `limit`
- `offset`

返回：

- 事件列表
- 分页信息
- 批注摘要

### `GET /api/events/{event_id}`

返回单条记录详情：

- 人物
- 年月
- 皇帝
- 年号
- 原文
- 事件类型
- 批注
- Excel 源单元格

### `GET /api/people`

人物检索与自动补全。

参数：

- `q`

返回：

- 标准名
- 别名
- 事件数
- 首次出现年份
- 末次出现年份

### `GET /api/people/{person_id}`

人物详情。

返回：

- 全部事件
- 任期状态记录
- 时间范围

### `GET /api/facets`

返回筛选面板枚举数据：

- 皇帝
- 年号
- 事件类型
- 年份范围
- 月份

### `GET /api/timeline`

按年月聚合事件数量，用于前端时间分布条。

### `GET /api/health`

健康检查。

数据库不存在时，返回清晰提示，指导先运行导入命令。

## 前端设计

采用三栏综合检索工作台布局。

### 左侧筛选面板

包含：

- 人名
- 年份范围
- 月份
- 皇帝
- 年号
- 事件类型
- 原文关键词
- 查询与重置按钮
- 北宋、南宋、任命、罢免等快速筛选

### 中间结果区

包含：

- 结果数量
- 当前筛选摘要
- 按年月聚合的事件分布条
- 可排序结果表格
- 分页
- CSV 导出

结果表格列：

- 公元年月
- 皇帝
- 年号
- 人物
- 事件类型
- 原文摘要

### 右侧详情面板

点击结果行后展示：

- 完整原文
- 批注
- 源单元格
- 人物全记录入口

### 人物详情视图

展示：

- 人物事件时间线
- 任期状态记录
- 首次与末次出现时间

## 视觉风格

默认采用深色档案风。

设计原则：

- 炭黑背景。
- 暖白正文。
- 金褐色强调。
- 低饱和分隔线。
- 表格行 hover 高亮。
- 当前选中行与右侧详情联动。
- 原文阅读区使用较高行距。
- 不使用过重仿古纹理，避免影响查询效率。

该风格应有档案检索气质，但仍以长期阅读、表格扫描和检索效率为优先。

## 错误处理

- API 参数非法时返回 422 和字段级错误。
- 数据库不存在时提示先运行导入脚本。
- 导入遇到无法解析的文本时不中断，写入 warning。
- 前端空结果时显示当前筛选条件和重置入口。
- 批注不存在时显示“无批注”，不隐藏追溯区域。

## 测试与验收

### 导入校验

验证：

- 人物列数量约 495。
- 职务变动记录约 1943；原始非空人物单元格约 3136，其中任期状态记录不入库。
- 批注数量约 113。
- 时间行、公元年、月份、皇帝、年号正确继承或关联。
- 导入警告写入审计表。

### API 测试

验证：

- 按年份、月份、人名、事件类型、皇帝、年号、关键词查询。
- 分页。
- 空结果。
- 非法参数。
- 数据库不存在提示。
- 事件详情。
- 人物详情。

### 前端验收

验证：

- 筛选区、结果表格、详情面板联动。
- 结果点击后显示原文、批注和源单元格。
- 深色档案主题在桌面和窄屏下不遮挡、不溢出。
- CSV 导出可用。

### 核心验收标准

- 能从 `宋代宰辅编年录.xlsx` 一键重建 SQLite 数据库。
- 能按公元年月和人名查询任免与任期记录。
- 能按皇帝、年号、事件类型和原文关键词进一步筛选。
- 每条结果都能追溯到 Excel 源单元格和原始文本。
- 自动解析不覆盖原文。
- 本地运行后能通过浏览器访问查询页面。

## 非目标

- 不把农历日自动换算为公历日。
- 不承诺完全自动解析“自何职、为何职、兼何职、罢何职”的完整变化链。
- 不建设正式多人数据库或权限系统。
- 不替代人工史料校订。
